理解原型继承中__proto__与prototype的关系

发表于 2016-09-22   |   分类于 javascript   |  
  1. 当设定Foo.prototype时，Foo.prototype.__proto__ -> Object.prototype。
  2. 当设定foo = new Foo()时，foo.__proto__ -> Foo.prototype。
  3. 所以foo.__proto__.__proto__ -> Foo.prototype.__proto__ -> Object.prototype。

但是原型继承时会发出什么呢？
下面三种实现原型继承：

  1. Object.setPrototypeOf(obj, proto) 将一个指定的对象的原型proto设置为另一个对象obj
  2. Object.create(proto) 创建一个拥有指定原型proto的对象
  3. new实现，具体见代码
  1. 当设定原型继承时，Bar.prototype.__proto__ -> Foo.prototype。
  2. 当设定bar = new Bar()时，bar.__proto__ -> Bar.prototype。
  3. 所以bar.__proto__.__proto__ -> Bar.prototype.__proto__ -> Foo.prototype。
  4. 所以bar.__proto__.__proto__.__proto__
    -> Bar.prototype.__proto__.__proto__
    -> Foo.prototype.__proto__
    -> Object.prototype。

读「你不知道的javascript」之prototype屏蔽

发表于 2016-09-20   |   分类于 javascript   |  

Object.create(proto, [ propertiesObject ])可以创建一个指定原型proto，和新的键和值
prototype.isPrototypeOf(obj) 返回prototype是否为obj的原型链上
obj.hasOwnProperty(prop) 返回obj是否含有指定的自身属性。

属性设置和屏蔽

  1. 原型链上层的普通属性，屏蔽

  2. 原型链上层的只读属性，赋值被忽略（严格模式抛出错误），不屏蔽

  3. 原型链上层的setter属性，赋值赋值时调用这个setter，不屏蔽

读「你不知道的javascript」之object描述符

发表于 2016-09-19   |   分类于 javascript   |  

获得属性描述符

Object.getOwnPropertyDescriptor(obj, prop)
获取某个obj的prop属性的描述符

所以默认下writable,enumerable,configurable均为true

设置属性描述符

Object.defineProperty(obj, prop, descriptor)
新建或修改某个obj的propp属性的描述符descriptor

writable

writable决定属性值是否可修改

如果在严格模式下还会抛出一个TypeError错误

configurable

configurable决定属性描述符是否可修改，这是不可逆的
例外: 可以将writable从true改成false

除此之外configurable:false，会禁止delete这个属性

enumerable

enumerable决定属性是否可枚举

Object.prototype.propertyIsEnumerable(prop)返回此prop是否可枚举（即返回其enumerable）

Object.keys(obj)返回此obj自身的可枚举属性名组成的数组
Object.getOwnPropertyNames(obj)返回此obj自身的可枚举与不可枚举属性名组成的数组

不变性

常量属性

不可修改，定义，删除的属性

禁止扩展

Object.preventExtensions(obj) 禁止添加新属性
Object.isExtensible(obj) 返回是否可扩展

密封

Object.seal(obj) 调用Object.preventExtensions(obj)并使所有属性configurable:false
Object.isSealed(obj) 返回是否密封

冻结

Object.freeze(obj)调用Object.seal(obj)并使所有属性writable:false
Object.isFrozen(obj) 返回是否冻结

get和set

当设置get或set时，会忽略value和writable
有以下两种方式设置get或set，其中第二种默认enumerable: false, configurable: false

一般来说get 和 set 同时出现

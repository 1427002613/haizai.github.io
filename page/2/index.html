demo——jQuery实现日历UI

发表于 2016-09-21   |   分类于 demo

  • demo地址：jQuery实现日历UI

  • 源码：github源码

  • 要求：百度ife2016任务41

  • 特点：

  1. jQuery实现，没有使用现成库或框架。
  2. 使用了javascript的构造函数模式及原型模式，只用了function Calendar(),Calendar.prototype={},new Calendar(),没有暴露其他变量。
  3. 上个月最后几天和下个月前几天计算得到，所以每月5行或6行不固定。
  4. 点击某天则active某天，上个月最后几天或下个月前几天，则先翻到那个月，再active。

  • 感想与总结：

  1. 第一次用jQuery完整的写一个组件，问题还是很多啦~~
  2. 整个流程：先init(),init最后执行render(),render最后绑定click()事件,click中lastMouth()和nextMouth()时再绑定render()。
  3. 计算日期过程调试非常多才成功，具体实现是这样：根据上个月最后一天的周几确定上个月还有几天，根据这个最后一天确定这个月，根据这个月最后一天的周几确定下个月初有几天，最后把这三部分合在在一起，每次render()时执行。
  4. 循环绑定耦合度高，不易维护。越发感觉到MVVM的好~

demo——原生javascript实现树形组件

发表于 2016-09-19   |   分类于 demo

  • demo地址：原生javascript实现树形组件

  • 源码：github源码

  • 要求：百度ife2016任务25

  • 特点：

  1. 原生html,css,javascript实现，没有使用其他库或框架。
  2. 使用了javascript的构造函数模式及原型模式，所有的节点都是new Tree(node)。
  3. 搜索时，将搜索到的值加红，如果那个节点是隐藏的，展开直到根。
  4. 用了点简单的css动画。

  • 感想与总结：

  1. 这是我第一次完整的写一个组件，问题很多啦~~
  2. 第一次用prototype，写起来很爽！对js和css的理解更深了。
  3. click事件的如果放在Tree.prototype里，new Tree时渲染有诡异的问题。
  4. 搜索值时用这种形式：外包裹innerHTML.replace(str,'<span style="...">'+str+'</span>')，重置时outerHTML=innerHTML，经常会出现bug，而且绑定在上面的click事件会消失/(ㄒoㄒ)/~~
  5. 在如此混杂的js添加css动画很晕，隐藏和删除的动画难以绑定。
  6. 用原生js操纵DOM真的很不方便（自从用了Vue以后），耦合非常高，代码不易维护（现在回头来看都快晕了）。
